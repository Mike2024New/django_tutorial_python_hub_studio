1. Создание/загрузка фикстуры:
Фикстура - сохранение модели (данных из БД) в формате json. Их можно загружать в приложение.
В видеокурсе при создании фикстуры у автора нет проблем с кодировкой. В моем же случае при создании фикстуры кодировка сбивается. 
Для решения этой проблемы создан файл fixture_encoder.py, который перекодирует фикстуры в формат utf-8.
    Для того, чтобы создать фикстуру (дамп модели в json), использовать:
        #примеры показаны на неком условном файле cats.json и модели Categories приложения goods
        1. Команда в терминале:
            "python manage.py dumpdata goods.Categories > fixtures/goods/cats.json"
        2. Применить скрипт из fixture_encoder. он переведет кодировку json в utf-8 и все будет корректно отображаться
    Для чтения фикстуры в БД:
        1. Команда в терминале:
            "python manage.py loaddata fixtures/goods/cats.json"

2. У моделей можно определять self методы, которые будут доступны в html шаблоне. (Эти методы нужны для преобразования данных)

3. Для того чтобы получать данные из БД (модели) не в рамках контроллеров, внутри приложения создаётся папка "templatetags" и внутри этой папки
нужно файл __inint__.py, а затем уже создать файл с функциями в которых будут храниться orm запросы в БД. За тем в базовом html документе 
(base.html), нужно загрузить этот файл с функциями {% load goods.tags %}, и внутри самого шаблона обратиться к нужной функции например 
{% tag_categories as categories %}, т.е. взяли данные из функции tag_categories. Это позволяет брать данные напрямую в обход контроллеров (views.py). 
Что позволяет централизовано управлять логикой запросов.

4. Динамические url обеспечиваются за счёт конверторов прописанных в маршрутах urls.py приложения (например приложение goods), пример такого url в 
файле urls.py 
    path('product/<slug:product_slug>/',view=views.product,name='product')
в этом пути прописано, что при обращении по пути 'product/..' будет выполнен вызов метода product контролера view, а тот контролер в свою очередь
принимает параметр записанный после пути "product" и выполняет запрос в БД и возвращает полученные данные (точнее карточку самого товара).
